# Talos.GameLLMBench
A benchmark framework for evaluating LLM capabilities in game development.

当前正式题库为连续编号的1000道单选题，范围为Q001-Q1000。

#### 评分标准详解
- **基础得分**: 每答对一题得 1 分，答错或格式错误 0 分。
- **答案格式**: 基准格式为 `Q001. A` 这类“题号 + 单个大写字母”输出；评分脚本兼容 `Q001. A | 分析内容` 作为扩展格式，但不要求必须附带分析。
- **维度评级**:
  - `> 85%` : 专家级 (具备直接参与核心架构与性能优化的能力)
  - `70% - 85%`: 高级 (具备独立排查复杂 Bug 与业务开发的能力)
  - `50% - 70%`: 初中级 (熟悉常用 API，但深层原理薄弱)
  - `< 50%` : 入门级 (存在大量知识幻觉，不可信)

1. **生成题目 (人类/高级模型完成)**: 参考 `gen_questions_rule.md` 生成题库。为了保证稳定性和防止大文件导致模型输出截断，题库被拆分为 10 个子文件存放在 `questions/` 目录下（如 `questions_01_core.md`）。
2. **生成答案密钥**: 将对应分片的标准答案写入 `answer_key/` 目录（严禁将此目录暴露给被测模型）。
3. **被测模型答题**: 将 `questions/` 下的题目通过 `llm_prompt.md` 喂给被测大模型，并将模型的输出结果按批次保存到 `answers/` 目录下（推荐命名如 `llm_answer_01_core.txt`，兼容旧命名 `llm_answer_01.txt`）。
4. **运行评分**:
   直接运行一键评分脚本，它会自动聚合所有的题目、密钥和答案，输出最终的全局成绩：
   ```bash
   bash run_score.sh
   ```
